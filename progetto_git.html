CHE COS'E' GIT

GIT è un sistema di software di controllo di versione distribuito, il che vuol dire che permette di tenere traccia delle modifiche apportate al codice sorgente senza dover usare un server centrale. Grazie a questa "feature" vari progetti software usano GIT per tale attività. GIT serve principalmente come version control system (controllo di versione distribuito) il quale può servire nel caso di lavoro allo stesso progetto da parte di due o più sviluppatori ed è possibile visualizzare chi ha cambiato qualcosa, quando e perchè, oppure se per caso viene cancellato qualcosa si riesce a recuperare ciò che prima, senza usare GIT, si poteva considerare perduto.


STORIA DI GIT

Come per molte grandi cose della vita, Git è nato con un po' di distruzione creativa e polemiche infuocate. Il kernel di Linux è un progetto software open source di grande portata. Per buona parte del tempo (1991-2002) della manutenzione del kernel Linux le modifiche al software venivano passate sotto forma di patch e file compressi. Nel 2002, il progetto del kernel Linux iniziò ad utilizzare un sistema DVCS proprietario chiamato BitKeeper. Nel 2005 il rapporto tra la comunità che sviluppa il kernel Linux e la società commerciale che aveva sviluppato BitKeeper si ruppe, e fu revocato l'uso gratuito di BitKeeper. Ciò indusse la comunità di sviluppo di Linux (e in particolare Linus Torvalds, il creatore di Linux) a sviluppare uno strumento proprio, basandosi su alcune delle lezioni apprese durante l'utilizzo di BitKeeper. Alcuni degli obiettivi del nuovo sistema erano i seguenti:
  • Velocità
  • Design semplice
  • Ottimo supporto allo sviluppo non-lineare (migliaia di rami paralleli)
  • Completamente distribuito
  • Capacità di gestire, in modo efficiente (velocità e dimensione dei dati), grandi progetti come il kernel Linux
Fin dalla sua nascita nel 2005 Git si è evoluto e maturato per essere facile da usare e tuttora mantiene le sue qualità iniziali. È incredibilmente veloce, è molto efficiente con progetti grandi e ha un incredibile sistema di ramificazioni, per lo sviluppo non lineare.

VANTAGGI NELL'USARE GIT

I principali vantaggi di GIT riguardante l'uso da parte degli sviluppatori o da parte dei semplici utenti sono:
  • Possibilità di tenere conto di una "libreria" dove è possibile tenere traccia dei file che sono stati modificati nel tempo
  • Possibilità di confrontare quei file, e magari poter ripristinare una versione precedente
  • Possibilità di creare una fusione di uno o più versioni dello stessco codice sorgente
  • Possibilità di lavorare in diversi parti del codice sorgente da più sviluppatori senza rovinare il tutto. Avendo la possibilità di usare rami differenti
  • Quando vengono sviluppati nuovi codici per il codice sorgente si ha la possibilità di fonderli insieme senza preoccuparsi se qualcosa viene cancellato dato che esiste uan "cronistoria" dove è possibile ritornare indietro


COME FUNZIONA GIT



Git lavora con una serie di comandi, i quali permettono agli sviluppatori di utilizzare appieno tutto il potenziale di GIT Andiamo a vedere nel dettaglio i comandi principali:

  • CREARE LA REPOSITORY PER UN PROGETTO
    • Eseguendo il comando "git init" si andrà a creare il repository locale, dove noi lavoreremo
  • AGGIUNGERE I FILE DA INSERIRE NELLA VERSIONE SUCCESSIVA
    • eseguendo il comando "git add" si aggiungono tutti i file, vengono riconosciuti altri comandi per aggiungere ed aggiungono nello specifico delle singole cartelle o dei determinati file rispettivamente sono ("git add nomeCartella" ed "git add percorso/nomefile")
  • VISUALIZZARE LO "STATO" DEL CODICE
    • Eseguendo il comando "git status" si visualizza lo "stato" del codice, che viene diviso in 3 sezioni: i file che sono stati aggiunti tramite "add" rispetto la versione precedente, i file modificati ma che non verranno aggiunti nella prossima versione, ed infine mostra i file che non sono mai stati inseriti nel controllo versione.
  • VISUALIZZARE LE DIFFERENZE E LE MODIFICHE EFFETTUATE SUI FILE
    • Eseguendo il comando "git diff" , come suggerisce il nome, potremo andare a vedere le differenze presenti tra due versioni del codice differenti.
  • INVIARE LE MODIFICHE
    • dopo aver aggiunto i file è possibile effettivamente "inviare" tali modifiche eseguendo il comando "git commit".
  • VISUALIZZARE LA CRONOLOGIA
    • E' possibile visualizzare la cronologia dei commit eseguendo il comando "git log"
  • CREARE UN BRANCH
    • Eseguendo il comando "git checkout -b nomeDelBranch" viene creato un nuovo branch, chiamato "nomeDelBranch", tutti i comandi di verifica effettuati si riferiranno al nuovo branch. Se si vuole tornare al ramo principale basta eseguire il comando "git checkout master".
  • FONDERE UN RAMO SECONDARIO (nomeBranch) CON UN RAMO PRINCIPALE (master)
    • Per poter fondere i due rami bisogna: essere sul master e usare il comando "git merge nomeDelBranch".
